The CSS Position Property

How is the CSS position property used?

HTML elements are, by default, positioned in the order that they appear in the HTML document. In the "flow" is one term. Element layout may be modified and elements may even be removed from the flow using one or more of the CSS positioning properties. Examples include the display property and the float property but our focus for now is the position property.

The top, bottom, left and right properties are used to specify a position for an element. However, these properties are only applicable for certain values of the position property.

What are the different values for the CSS position property? How do they work?

The CSS position property can have one of several different values.

position: [static | relative | absolute | fixed | sticky];

Elements using the "absolute" or "fixed" values for the display property are considered to use absolute positioning and are removed from the flow. Elements using the "relative" value for the display property are considered to be relatively posisioned and are not removed from the flow.

  • static

    This is the default value for the position property. The element is positioned in the normal flow of elements on the HTML page.

    The top, bottom, left and right properties do not apply to static elements.

  • relative

    The top, bottom, left and right properties are used to position the element relative to its normal position in the flow.

    Other elements in the same container are positioned to leave space for the relatively-positioned element as though it were in its normal position in the flow. The position of other elements is not adjusted to fill in any gaps that may result from the relative positioning.

  • absolute

    The top, bottom, left and right properties are used to position the element relative to its closest positioned ancestor. If the absolute-positioned element has no positioned ancestors then it is positioned relative to the initial containing block (e.g., the document body).

    Other elements in the same container are positioned as though the absolute-positioned element were not present. The absolute-positioned element is removed from the flow.

  • fixed

    The top, bottom, left and right properties are used to position the element relative to the viewport.

    Other elements are positioned as though the fixed-positioned element were not present. The fixed-positioned element is removed from the flow.

  • sticky

    Sticky positioning is a hybrid of relative and fixed positioning. The element is treated as relative positioned until it crosses a specified threshold, at which point it is treated as fixed positioned. MDN

    To make sticky positioning work, a threshold must be specified using one or more of the top, bottom, left and right properties. The element will be relatively positioned until the threshold is reached (e.g., via the user scrolling the viewport) at which point the element will be positioned as fixed.

Overlapping Elements

Any positioned element may overlap other elements. The z-index property can be used to specify which elements appear over other elements. Elements are displayed in increasing order by z-index value so that the highest z-index value will be displayed last and therefore on top of other elements. A z-index property value may be negative.
