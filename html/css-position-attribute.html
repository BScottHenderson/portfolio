The CSS Position Attribute

What are the different values for the CSS position attribute? How do they work?

The CSS position attribute can have one of several different values.

  • static

    This is the default value for the position attribute. The element is positioned in the normal flow of elements on the HTML page.

    The top, bottom, left and right attributes do not apply to static elements.

  • relative

    The top, bottom, left and right attributes are used to position the element relative to its normal position in the flow. Other elements are positioned to leave space for the relatively-positioned element as though it were in its normal position. They are not adjusted to fill in any gaps that may result from the relative positioning.

  • absolute

    The top, bottom, left and right attributes are used to position the element relative to its closest positioned ancestor. If the absolute-positioned element has no positioned ancestors then it is positioned relative to the initial containing block (e.g., the document body). Other elements are positioned as though the absolute-positioned element were not present.

  • fixed

    The top, bottom, left and right attributes are used to position the element relative to the viewport. Other elements are positioned as though the fixed-positioned element were not present.

  • sticky

    Sticky positioning is a hybrid of relative and fixed positioning. The element is treated as relative positioned until it crosses a specified threshold, at which point it is treated as fixed positioned. MDN

Overlapping Elements

Any positioned element may overlap other elements. The z-index property can be used to specify which elements appear over other elements. Elements are displayed in increasing order by z-index value so that the highest z-index value will be displayed last and therefore on top of other elements. A z-index attribute value may be negative.
